# Anti-Bleed Protocol

## Purpose
Keep Grimverse visually and narratively isolated from any future series.

## Rules
- One series = one style lock
- One protagonist identity language
- One faction naming convention
- One world rule stack
- One review rubric
- One export pipeline

## Branching Policy
If a second series is introduced, it must receive:
- a new series id
- a new style lock
- a new world state file
- separate factions
- separate arcs and episodes
- separate export folders

## Current Active Branch
Grimverse only
